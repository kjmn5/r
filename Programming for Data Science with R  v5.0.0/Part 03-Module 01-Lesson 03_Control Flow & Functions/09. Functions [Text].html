09. Functions [Text]

Functions

Introduction

In the previous video, you had your first introduction to functions. Functions are one of the most powerful tools for any programmer, as they allow you to build a general solution that can be used.

Although you haven't been writing your own functions up to this point in the class, you have been using functions that someone else wrote.

Just a few of the many examples of functions you have used up to this point include:

  1. summary() - takes in a dataframe as an argument and returns a summary
  2. dim() - takes in a dataframe as an argument and returns the** number of rows and columns**
  3. head() - takes in a dataframe as an argument and returns the** first few rows**

Defining Your Own Functions

You saw in the previous video syntax that would allow you to create your own functions. Specifically, there is a keyword function.

If we want to create a function that always returns "Hello", we can create this in the following way:

sayHello = function(){
    return("Hello")
}

Whatever is beside return will be given back when the function is run. In order to run the function, you simply use the name provided on the left side of the equal sign with parentheses following as shown below.

[Input]:  sayHello()
[Output]: "Hello"

Arguments

However, the real power of functions is created when you introduce arguments. Arguments are the values that are provided to the function.

For our sayHello function, we might want to be specific about who we are saying hello to, and we could introduce an argument that would allow for this specificity as shown here:

sayHello = function(to.who){
    result = paste("Hello", to.who, sep=" ")
    return(result)
}

We can then use this function to say hello to my friend "Mohit".

[Input]:  sayHello("Mohit")
[Output]: "Hello Mohit"

Default Arguments

In many cases, we provide the arguments of functions with default values. These values are used when no value is provided, but you can also write over the default values with your own argument values.

Let's look at this in practice in the below function, where I place **Mohit **as the default argument. You can see this with the to.who = "Mohit" line when creating the function definition.

sayHello = function(to.who = "Mohit"){
    result = paste("Hello", to.who, sep=" ")
    return(result)
}

This means that if we are saying hello specifically to Mohit, we don't need to provide a value at all in the to.who argument. See the result below using this new default.

[Input]:  sayHello()
[Output]: "Hello Mohit"

However, we can also overwrite with a new name.

[Input]:  sayHello("Jennifer")
[Output]: "Hello Jennifer"
