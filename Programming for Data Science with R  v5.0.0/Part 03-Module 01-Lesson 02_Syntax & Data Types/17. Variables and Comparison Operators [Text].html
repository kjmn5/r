17. Variables and Comparison Operators [Text]

Variables

Variables are created by using an = value. Variables allow us to store the result into a name and then use this result in other places.

For example, I can create the variable y

y = 4

Or I can create the variable z

z = 5

Once these variables are created, I can use them to calculate new values. For example:

[Input]:  y + z
[Output]: 9
[Input]:  y - z
[Output]: -1
[Input]:  y * z
[Output]: 20
[Input]:  y / z
[Output]: 0.8

Variable Naming

Creating names that are informative can help others read your code. There is always a balance between creating short variable names (that are fast to type) and use variable names that are easy for a reader of your code to understand.

Comparison Operators

There are a number of comparison operators in R. Basically anything you can think of! Below you can see an example of a number of them in comparing our variables y and z.

[Input]:  y = 4 # Create y variable
[Input]:  z = 5 # Create z variable

[Input]:  y > z
[Output]: FALSE

[Input]:  y < z
[Output]: TRUE

[Input]:  y <= z # less than or equal
[Output]: TRUE

[Input]:  y >= z # greater than or equal
[Output]: FALSE

[Input]:  y == z # exactly equal to one another
[Output]: FALSE

[Input]:  y != z # not equal to one another
[Output]: TRUE

You can also do these calculations on an entire vector all in one operation!

[Input]:  heights = c(67, 74, 63, 70)
[Input]:  heights >= 70 # compares all values in heights to 70
[Output]: FALSE TRUE FALSE TRUE

any, all, which, subset

any

There are some additional operators that can be quite helpful when working with vectors.

The any** **function returns **TRUE **if any of the values in a vector meet some criteria. For example, the below looks at all items in heights to see if they are less than or equal to 70. If even one of the values is less than 70, **TRUE **is returned.

In this case, 67, 63, and 70 meet the criteria, so we get a return of TRUE.

[Input]:  any(heights <= 70)
[Output]: TRUE

Alternatively, the following returns FALSE, because none of the heights are greater than 80.

[Input]:  any(heights > 80)
[Output]: FALSE

all

The all command is very similar to the previous command but only returns **TRUE **if all of the elements of a vector meet the provided criteria. So comparing the two, we can see a case where they differ since not **all **elements in heights meet the criteria, but some do. 

[Input]:  any(heights <= 70)
[Output]: TRUE
[Input]:  all(heights <= 70)
[Output]: FALSE

However, if we change the criteria, we see a new result.

[Input]:  all(heights > 60)
[Output]: TRUE

which

The which command returns the indices of the vector elements that meet a particular criteria. **Note that R indexes starting at 1. **Depending on your experience with other programming languages, you might be used to languages that use 0 indexing - R doesn't do this.

See below the indices of heights that are less than or equal to 70. We can also pull the values that are exactly equal to 70 or not equal. 

[Input]:  which(heights <= 70)
[Output]: 1 3 4
[Input]:  which(heights == 70)
[Output]: 4
[Input]:  which(heights != 70)
[Output]: 1 2 3

subset

The subset command works a lot the same way as the which command with the largest difference being that a subset command will allow you to pull the indices from other vectors.

For example, if you want to know the eye colors associated with people who are taller than 70 inches, we would use the subset command.

# four people with 1st person eye and height in index 1
# 2nd person has eye and height in index 2
# and so on...

[Input]:  eye.colors = c('blue', 'green', 'brown', 'brown')
[Input]:  heights = c(67, 74, 63, 70)

[Input]:  subset(eye.colors, heights > 70)
[Output]: 'green'

You can also use subset the same way you used which, but the above shows a scenario when you might prefer to use this command.
